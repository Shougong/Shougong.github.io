Transformer模型实现(一)

2021-01-08
Word count: 4.7k   |   Reading time≈ 21 min

写在开头

最近在做实验的时候，发现复现的一个模型在误分类上存在着attention过多考虑一些多次出现的词的情况，导致模型在捕捉关键特征上性能比较欠缺，实验后对模型进行分析，可能是引入concept后，因为attention是对整个concept构成的文本序列做的，不同entity对应的concept混合在一起后，会造成比较严重的歧义，合理的做法应当是让概念词仅作用于对应的entity，这样子会有更好的效果，此外，由于entity可能是多义的，比如"apple"，在现代语义中，有苹果（水果）和苹果公司两个含义在其中，因此需要做区分，直接使用attention可能忽视了原始文本中的上下文语义关系，最后做出误判，因此，选择合适的concept可能需要借助mask机制来实现，为此，记录下学习transformer实现的代码，本门的版本是借鉴了哈工大SCIR实验室翻译的哈佛大学的复现版本[1]。

模型简介

Figure 1是Transformer模型的整体架构图，遵循标准的Encoder-Decoder结构，这一架构的前身源于Seq2Seq + Att模型，首先看左边的Encoder部分，Encoder编码器由embedding层、位置编码（Positional Encoding）和N层（论文采用了6层）子层组成（特征提取层，这样的子层后来被叫做transformer层）。右边的Decoder解码器部分与Encoder部分在结构上比较接近，也是由embedding层、位置编码和N层子层组成。对于不同的应用，embedding层所使用的预训练向量是不同的，可能是但是这篇文章仅仅是介绍transformer模型的实现，因此，使用相同的向量空间。

Encoder

Encoder将输入序列(x1,...,xn)(x_{1}, ..., x_{n})(x1​,...,xn​)映射到一个连续表示序列z=(z1,...,zn)z=(z_{1}, ..., z_{n})z=(z1​,...,zn​)。观察Figure 1并结合论文[2]，Encoder由N=6层EncoderLayer子层构成，每个子层又可以拆分成三个部分：多头注意力机制（一种self-attention）、残差连接和归一化层以及一个Feed Forward前馈神经网络（全连接层）。

这一部分的残差连接思想来源于ResNet网络[3]，使用跳连接的方法将前层的特征得以传递到当前层，使得梯度可以顺利传播。ResNet也因其简单高效，堪称近几年深度学习领域的重大工作之一。

层归一化

层归一化实际上就是对每一层的神经元做归一化操作，为什么需要这个操作呢？是因为模型在训练过程中，每一层的参数在更新过程中，会改变下一层输入的分布，也就是发生了内部协变量偏移，神经网络层数越多，表现得越明显，这是因为模型层数越深，底部的微小偏移会在高层产生很大的影响，与"蝴蝶效应"有着异曲同工之妙。上面层归一化的实现采用了标准正态分布，模型初始化中的features对应的是神经网络参数，详细的介绍可以看这里[4]。

Decoder

Decoder将Encoder产生的连续表示序列z=(z1,...,zn)z=(z_{1}, ..., z_{n})z=(z1​,...,zn​)进行解码，生成完整的输出序列y=(y1,...,yn)y=(y_{1}, ..., y_{n})y=(y1​,...,yn​)。对于每一步解码，模型都是自回归的，即在生成下一个符号时将先前生成的符号作为附加输入。

Decoder与Encoder一样，也采用了N=6个相同层组成 ，除了每个编码器中的两个子层以外，解码器还插入了第三种子层对编码器的输出实行多头Attention，其余的与Encoder是类似的，都是用了残差连接、层归一化处理。这里就不进行赘述了，后面主要讲一下Attention机制的实现。

Attention机制

在Transformer模型里，使用了两种attention机制，一种是逐点注意力机制，另一种是多头注意力机制，都是self-attention的变体。

Scaled Dot Product Attention

Attention函数可以将Query和一组key-value对映射到输出，其中Query、Key、Value和输出都是向量。输出是值的加权和，其中分配给每个value的权重由query与相应的key的兼容函数计算，这种Attention机制称为"Scaled Dot Product Attention"。该函数的输入包含维度为dkd_{k}dk​的query和key，以及维度为dvd_{v}dv​的value。该函数首先分别计算query与各个key的点积，然后将每个点积除以dk\sqrt{d_{k}}dk​​，最后使用softmax函数获得key的权重，对应具体公式：

 Attention (Q,K,V)=softmax⁡(QKTdk)V\text { Attention }(Q, K, V)=\operatorname{softmax}\left(\frac{Q K^{T}}{\sqrt{d_{k}}}\right) V  Attention (Q,K,V)=softmax(dk​​QKT​)V

这里在阅读这篇文章的时候，也补充了一些新知识。在Attention机制中，除了点积Attention还有加和Attention，这种Attention使用具有单个隐藏层的前馈网络来计算兼容函数。两种Attention在理论上复杂度相似，但在实践中，点积Attention计算得更快，也更节省空间，因为可以使用高效的矩阵乘法来实现。

这两种机制在dkd_{k}dk​较小的时候，表现相似，但在不放缩较大的dkd_{k}dk​时，加和Attention要优于点积Attention。文章怀疑，对于较大的dkd_{k}dk​，点积大幅增大，将softmax函数推向具有极小梯度的区域，因为，假设qqq和kkk是独立的随机变量，平均值为0，方差为1，这样他们的点积为q⋅k=∑i=1dkqikiq \cdot k=\sum_{i=1}^{d_{k}} q_{i} k_{i}q⋅k=∑i=1dk​​qi​ki​，很显然，该结果均值为0，方差为dkd_{k}dk​，而加和Attention，均值为0，方差为2。为了抵消这种影响，文章使用了1dk\frac{1}{\sqrt{d_{k}}}dk​​1​来缩放点积。

MultiHead Attention

"多头"机制能让模型考虑到不同位置的Attention，另外，"多头"Attention可以在不同的子空间表示不一样的关联关系，使用单个Head的Attention一般达不到这种效果，该机制遵循下述公式：

 MultiHead (Q,K,V)= Concat ( head 1,…, head h)WO where head i= Attention (QWiQ,KWiK,VWiV)\begin{array}{l}\text { MultiHead }(Q, K, V)=\text { Concat }\left(\text { head }_{1}, \ldots, \text { head }_{\mathrm{h}}\right) W^{O} \\\text { where head }_{\mathrm{i}}=\text { Attention }\left(Q W_{i}^{Q}, K W_{i}^{K}, V W_{i}^{V}\right)\end{array}  MultiHead (Q,K,V)= Concat ( head 1​,…, head h​)WO where head i​= Attention (QWiQ​,KWiK​,VWiV​)​

文章使用了h=8h=8h=8个Head并行的Attention，对每一个Head来说，dk=dv=dmodel/h=64d_{k}=d_{v}=d_{model}/h=64dk​=dv​=dmodel​/h=64，总计算量与完整维度的单个Head的Attention很接近。

多头Attention的本质就是多个Scaled Dot Product Attention。

Attention在模型中的应用

Transformer中以三种不同的方式使用了“多头”Attention：

  • 在"Encoder-Decoder Attention"层，Query来自先前的解码器层，并且Key和Value来自Encoder的输出。Decoder中的每个位置Attend输入序列中的所有位置，这与Seq2Seq模型中的经典的Encoder-Decoder Attention机制[15]一致。

  • Encoder中的Self-attention层。在Self-attention层中，所有的Key、Value和Query都来同一个地方，这里都是来自Encoder中前一层的输出。Encoder中当前层的每个位置都能Attend到前一层的所有位置。

  • 类似的，解码器中的Self-attention层允许解码器中的每个位置Attend当前解码位置和它前面的所有位置。这里需要屏蔽解码器中向左的信息流以保持自回归属性。具体的实现方式是在缩放后的点积Attention中，屏蔽（设为负无穷）Softmax的输入中所有对应着非法连接的Value。

Position-wise前馈网络

除了Attention子层以外，Encoder和Decoder中的每个层都包含一个全连接前馈网络，分别应用于每个位置，其中包括连个线性变换，然后使用ReLU作为激活函数。

FFN⁡(x)=max⁡(0,xW1+b1)W2+b2\operatorname{FFN}(x)=\max \left(0, x W_{1}+b_{1}\right) W_{2}+b_{2} FFN(x)=max(0,xW1​+b1​)W2​+b2​

虽然线性变换在不同位置上是相同的，但他们在层与层之间使用不同的参数，这其实是相当于使用了两个内核大小为1的卷积，这里设置输入和输出的维度为dmodel=512d_{model}=512dmodel​=512，内层的维度为dff=2048d_{ff}=2048dff​=2048。

Embedding

Transformer模型的Embedding层，与我们常用的Embedding是类似的，也可以使用预训练词向量，唯一不同的是，文章对所有向量乘了一个常数dmodel\sqrt{d_{model}}dmodel​​。

位置编码

由于模型不包含递归和卷积结构，Transformer模型为了有效利用序列的顺序特征，加入了序列中各个token间相对位置或token在序列中绝对位置的信息，并将位置编码到编码器和解码器底部的输入Embedding中。由于位置编码与Embedding具有相同的维度dmodeld_{model}dmodel​，因此两者可以直接相加，这里选择了不同频率的正弦和余弦函数：

PE(pos,2i)=sin⁡(pos/100002i/dmodel )PE(pos,2i+1)=cos⁡(pos/100002i/dmodet )\begin{array}{l}P E_{(p o s, 2 i)}=\sin \left(p o s / 10000^{2 i / d_{\text {model }}}\right) \\P E_{(p o s, 2 i+1)}=\cos \left(p o s / 10000^{2 i / d_{\text {modet }}}\right)\end{array} PE(pos,2i)​=sin(pos/100002i/dmodel ​)PE(pos,2i+1)​=cos(pos/100002i/dmodet ​)​

其中pospospos是位置，iii是维度，也就是说，位置编码的每个维度对应一个正弦曲线，波长遵循等比级数。而选择这个函数的原因是，模型在构造之初，假设该函数能让模型很容易学会Attend相对位置，因为对于任何固定的偏移量kkk，PEpos+kPE_{pos+k}PEpos+k​可以表示为PEposPE_{pos}PEpos​的线性函数，此外，还在位置编码中加入了Dropout，并设置dropout=0.1。
